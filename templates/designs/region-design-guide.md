# 区域详细设计指南（CanvasVideo Skill 分设计规范）

> 本指南是 AI 执行**单个区域详细设计**的规范。基于骨架设计，为每个区域规划组件、布局、样式和时间轴。
> 适用范围：骨架设计确认后，逐区域执行。

---

## 使用说明

1. **输入**：`design-skeleton.md` 中该区域的信息 + 前序区域的设计（保持连贯性）
2. **AI 执行**：按本指南"四阶段八步"，**逐步生成区域设计文档**
3. **最终输出**：`design-P{n}.md`（单个区域的完整设计）
4. **用户确认**：每个区域设计完成后，用户确认或修改

---

## 阶段总览

| 阶段 | 名称 | 步骤 | 产出 |
|------|------|------|------|
| 阶段一 | 内容规划 | 1-2 | 区域文案、信息层级 |
| 阶段二 | 布局设计 | 3-5 | ASCII 布局图、组件清单 |

## 阶段一：内容规划

### 步骤 1：区域文案

根据骨架设计中的区域主题，编写该区域的文案：

| 元素 | 内容 | 字数限制 |
|------|------|----------|
| 主标题 | 区域核心信息 | ≤ 15 字 |
| 副标题 | 补充说明 | ≤ 30 字 |
| 正文 | 详细内容（如有）| ≤ 100 字 |
| 标签/徽章 | 关键词 | ≤ 8 字 |

**原则**：
- 文案简洁，适合视频快速阅读
- 每区域最多 1 个主标题 + 2-3 个辅助元素

### 步骤 2：信息层级

确定信息的优先级和视觉权重：

```
层级 1（最重要）：主标题 — 最大字号、最醒目颜色
层级 2（重要）：副标题/核心数据 — 中等字号
层级 3（辅助）：标签、说明文字 — 小字号
层级 4（装饰）：图标、背景元素 — 不干扰阅读
```

## 阶段二：布局设计

### 步骤 3：ASCII 布局图

用 ASCII 绘制该区域在 viewport 内的布局：

```
┌─────────────────────────────┐
│                             │
│     【主标题】               │  ← TitleComponent
│                             │
│  ┌─────────────────────┐   │
│  │                     │   │
│  │    【核心内容】      │   │  ← ShockComponent / CardComponent
│  │                     │   │
│  └─────────────────────┘   │
│                             │
│       [标签]                │  ← BadgeComponent
│                             │
└─────────────────────────────┘
```

### 步骤 4：组件选择

根据内容选择合适的组件类型：

| 组件类型 | 适用场景 | 数量限制 |
|----------|----------|----------|
| TitleComponent | 主标题 | 每区域 1 个 |
| TextComponent | 正文、说明 | 每区域 ≤ 2 个 |
| ShockComponent | 强调数据、金句 | 每区域 ≤ 1 个 |
| BadgeComponent | 标签、分类 | 每区域 ≤ 2 个 |
| ImageComponent | 图片展示 | 每区域 ≤ 1 个 |
| CardComponent | 结构化信息 | 每区域 ≤ 2 个 |
| GraphicComponent | 图表、流程 | 每区域 ≤ 1 个 |

**原则**：单个区域组件总数 ≤ 5 个（保持简洁）

### 步骤 5：组件清单

列出该区域所有组件：

| ID | 类型 | 内容 | 位置 (x, y, w, h) |
|----|------|------|-------------------|
| P1-001 | TitleComponent | "AI时代，学习即提问" | (20, 30, 740, 70) |
| P1-002 | ShockComponent | "善用工具，持续迭代" | (120, 120, 540, 80) |
| P1-003 | BadgeComponent | "AI LEARNING" | (310, 220, 160, 40) |

## 阶段三：组件样式

### 步骤 6：customStyle 设计

为每个组件设计样式。必须先调用 API 查询组件规范：

```js
// 必须先调 API 获取组件规范
const { queryApi } = require('./scripts/query-api');
const specs = await queryApi.batchGetComponentSpec(['TitleComponent', 'ShockComponent']);
```

**样式设计原则**：
- 颜色与主题一致（从 `templates/themes/{theme}.json` 取色）
- 字号层级清晰（标题 > 正文 > 标签）
- 留白适当（组件间距 ≥ 20px）

### 步骤 7：视觉丰富度检查

检查区域是否满足视觉丰富度门槛：

| 门槛 | 要求 |
|------|------|
| 门槛 1 | 至少 1 个彩色元素 |
| 门槛 2 | 至少 2 种字号 |
| 门槛 3 | 至少 1 个装饰元素（阴影、圆角、渐变） |
| 门槛 4 | 图片/图标占位符已标注 |

## 阶段四：时间轴

### 步骤 8：时间轴设计

设计组件和字幕的时间安排：

**组件时间轴**：

| 组件 | start | end | 说明 |
|------|-------|-----|------|
| P1-001 | 0 | 5 | 全程显示 |
| P1-002 | 1.5 | 5 | 延迟出现 |
| P1-003 | 3 | 5 | 最后出现 |

**字幕时间轴**（仅配音模式）：

| 开始 | 结束 | 文字 |
|------|------|------|
| 0 | 2.5 | "AI时代，学习即提问" |
| 2.5 | 5 | "善用工具，持续迭代" |

**时间设计原则**：
- 组件入场有间隔（避免同时出现）
- 关键信息停留时间 ≥ 2 秒
- 字幕与对应组件同步

- [ ] 时间轴无重叠冲突
