# 视频总设计指南（CanvasVideo Skill 骨架设计规范）

> 本指南是 AI 执行视频**骨架设计**的规范。确定视频的整体结构：区域划分、时长分配、主题风格。
> 适用范围：**创作模式 / 口播模式** 下的首次视频设计。

---

## 使用说明

1. **用户提供需求**：项目主题/受众/时长/风格（创作模式）或 口播音频+SRT（口播模式）
2. **AI 执行**：按本指南"步骤 0 + 三阶段"，**生成分设计文档，不能跳过任何步骤**
3. **最终输出**：`design-skeleton.md`（骨架设计）+ 多个 `design-P{n}.md`（区域设计）
4. **用户确认**：确认骨架后，再逐区域进入详细设计

## 步骤 0：视频目标（项目元信息）

### 产出格式

| 字段 | 内容 |
|------|------|
| 模式 | 创作模式 / 口播模式 |
| 主题 | （用户提供） |

## 阶段一：内容理解

### 步骤 1：内容类型判定

根据主题和受众，确定视频的内容类型：
- **知识科普**：概念解释、原理解读
- **产品展示**：功能介绍、使用场景
- **案例分享**：实战经验、成果展示
- **观点表达**：行业见解、趋势判断

### 步骤 2：情绪曲线设计

绘制视频整体的情绪起伏：

```
情绪强度
  ↑
  │    ╱╲
  │   ╱  ╲    ╱╲
  │  ╱    ╲  ╱  ╲
  │ ╱      ╲╱    ╲___
  └────────────────────→ 时间
    开场   发展   高潮   结尾
```

### 步骤 3：区域划分

**核心决策**：将视频划分为几个区域？

| 视频时长 | 建议区域数 | 每区域时长 |
|----------|-----------|-----------|
| 5-15 秒 | 1-2 个 | 5-10 秒 |
| 15-30 秒 | 2-3 个 | 8-15 秒 |
| 30-60 秒 | 3-5 个 | 10-20 秒 |
| 60-120 秒 | 5-8 个 | 10-20 秒 |

**区域划分原则**：
1. 每个区域有明确的**主题或功能**（开场、痛点、方案、案例、CTA）
2. 区域之间有过渡或逻辑递进
3. 单个区域不超过 20 秒（保证 LLM 聚焦）

**产出**：区域划分表

| 区域 | 名称 | 时间范围 | 主题 | 情绪 |
|------|------|----------|------|------|
| P1 | 开场 | 0-8s | 吸引注意 | 好奇 |
| P2 | 痛点 | 8-20s | 问题呈现 | 共鸣 |
| P3 | 方案 | 20-40s | 解决方案 | 期待 |
| P4 | CTA | 40-60s | 行动号召 | 冲动 |

## 阶段二：视觉策略

### 步骤 4：区域规划

为每个区域确定：

| 属性 | 说明 |
|------|------|
| 区域位置 (x, y) | 在 canvas 上的坐标 |
| viewport 比例 | 16:9 / 9:16 / 1:1 等 |
| canvas 尺寸 | 根据区域数计算总宽度 |

**canvas 宽度计算**：
```
canvas.width = 区域数 × 单个区域宽度 + 间距
canvas.height = 根据内容类型确定（通常 600-1300）
```

### 步骤 5：配色与 BGM

**配色方案**：
- 从 `templates/themes/{theme}.json` 选择基础主题
- 根据品牌/风格调整主色、辅色

**BGM 选择**（仅 BGM 模式）：
- 从 `templates/bgm/bgm-catalog.md` 选择匹配风格的音乐
- 记录 `bgmStyle` 字段值
