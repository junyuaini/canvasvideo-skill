# 区域设计（口播模式）：P{n} — {主题}

> 口播模式：用户提供音频 + SRT 字幕，AI 负责画面设计。
>
> ⚠️ **核心原则**：字幕是系统自动显示的前提条件，画面是字幕的**视觉翻译**，不是重复。

## 示例：P2 — 四步搞定任何技能

**区域信息**

| 字段 | 值 |
|------|-----|
| 区域序号 | P2 |
| 区域类型 | Point |
| 时间段 | 6s - 14s |
| 时长 | 8秒 |
| 包含字幕 | 字幕3-4 |
| 情绪强度 | 中 |
| 区域位置 | 前1/3 |
| 前区域情绪 | 高 |
| 核心信息 | 四步搞定任何技能 |

**字幕内容**

| 字幕序号 | 时间段 | 内容 |
|---------|--------|------|
| 3 | 6.2-9.8 | 四步搞定任何技能 |
| 4 | 9.8-13.5 | 第一步：明确目标 |

**画面设计**

| 项目 | 值 |
|------|-----|
| 内容类型 | 步骤/流程 |
| 画面设计方向 | 数字"4"放大 + 步骤预览 |
| 推荐组件 | TitleComponent + GraphicComponent |

**布局与锚点**

- 布局模式：单点聚焦
- 锚点类型：数据锚点
- 锚点内容：数字"4"
- 锚点大小：占画面40%
- 锚点颜色：#00d4ff

> **注意**：锚点位置由前端自动流式布局，无需手动配置。

**组件清单**

> **注意**：组件位置由前端自动流式布局，无需手动配置。

1. ShockComponent（数字"4"） 大小：占40% 颜色：#00d4ff 延迟：0秒
2. TitleComponent（四步搞定任何技能） 大小：24px 颜色：#ffffff 延迟：0.5秒
3. GraphicComponent（步骤预览：①明确目标 ②找到最佳实践 ③快速试错 ④迭代优化） 大小：18px 颜色：#a0a0b0 延迟：1.0秒

**配色方案**

- 背景：#0F172A
- 主色：#ffffff
- 辅色：#a0a0b0
- 强调色：#00d4ff

**图片关键词**

- 图标：数字图标、步骤图标

**稳定期**

- 稳定期时长：0.8秒

**字幕与画面同步**

| 检查项 | 状态 |
|--------|------|
| 时间覆盖 | ✅ 6s-14s 覆盖字幕3-4 |
| 语义一致 | ✅ "四步"对应数字"4"，"技能"对应步骤预览 |
| 关键信息 | ✅ "四步"用ShockComponent强调 |

**设计意图**

字幕提到"四步搞定任何技能"，画面用放大的数字"4"作为视觉锚点，配合步骤预览，让观众一眼看懂核心信息。数字用强调色#00d4ff，与tech风格一致。
