# 模式与音频规则（Mode & Audio Rules）

> 视频生成模式的定义、音频字段写法、字幕共生强制规则、创作模式默认 BGM 行为。
> **本文档是 hard rule 单一来源**，被以下流程引用：
> - 主流程：SKILL.md「第一次交互」「打包上传前 validate」
> - 子流程：video_design_guide.md 步骤 0
> - 校验：云端 `/api/projects/validate`（`subtitles` 与 `audio` 共生由 schema dependencies 表达，本地不再做 schema 校验，B 方案）

---

## 1. 两种模式定义

| 模式 | 用户提供 | 字幕（subtitles） | AI 负责 |
|------|---------|-----------------|--------|
| **创作模式** | 主题/目标/时长/风格等文本信息 | ❌ **不生成字幕**（除非用户主动提供 SRT） | 自动生成画面、组件、占位素材；视频以图文为主 |
| **口播模式** | 口播音频（必须） + SRT 字幕（必须） | ✅ **必须有字幕**（与 SRT 一致） | 严格按音频/SRT 排版，自动生成画面素材 |

**默认行为**：
- 创作模式：`project.json` 中**不允许**出现 `subtitles` 数组；`audio` 字段允许且默认配 BGM（详见 §4）
- 口播模式：`project.json` 中**必须同时**有 `audio` 字段和 `subtitles` 数组

**口播模式输入校验**：
- 必须提供 `.mp3 / .wav / .m4a` 音频和 `.srt` 字幕
- 路径不存在或扩展名不支持时**阻塞进入第二阶段**，要求用户重新提供
- 路径包含 `..` 等穿越字符时拒绝拷贝

## 2. 音频字段写法

**核心原则**：系统不区分录音/BGM——`audio` 字段就是"一个音频文件"。**通过设置/不设置 `loop / fadeIn / fadeOut` 三个字段表达用法意图**。

```jsonc
{
  // 写法 1：字符串简写（向后兼容；视为"配音用法"）
  "audio": "./assets/voice.mp3",

  // 写法 2：对象形式（推荐）
  "audio": {
    "path": "./assets/bgm.mp3",
    "loop": true,        // ⭐ 仅 BGM 用法设置；配音用法保持默认 false
    "fadeIn": 1,         // ⭐ 仅 BGM 用法设置；配音用法保持默认 0
    "fadeOut": 2         // ⭐ 仅 BGM 用法设置；配音用法保持默认 0
  }
}
```

## 3. 用法判定与字幕共生（强制）

### 3.1 用法判定矩阵

| 场景 | audio 写法 | 是否配 SRT | 行为 |
|---|---|---|---|
| **配音用法**（口播 / TTS 解说） | 字符串 OR 对象只有 path | ✅ **必须** | **不设** loop/fadeIn/fadeOut |
| **BGM 用法**（背景音乐） | 对象，含 loop=true | ❌ **严禁** | **必须设** `loop: true, fadeIn: 1, fadeOut: 2` |

### 3.2 判定流程

```
用户提供了 SRT 字幕？
├─ 是 → 配音用法 → audio: "./xxx.mp3"
│       不设 loop/fadeIn/fadeOut，subtitles 必填
│
└─ 否 → BGM 用法 → audio: { path, loop:true, fadeIn:1, fadeOut:2 }
        不设 subtitles 数组
```

### 3.3 字幕与音频共生（schema dependencies 强约束）

| audio 用法 | subtitles 字段 | 是否允许 | 说明 |
|---|---|---|---|
| 配音用法（不设 loop/fade） | ✅ 有 | ✅ 允许 | 标准口播模式 |
| BGM 用法（设了 loop/fade） | ❌ 无 | ✅ 允许 | 标准 BGM 模式 |
| 无 audio | ❌ 无 | ✅ 允许 | 纯静音创作模式（不推荐，详见 §4） |
| 配音用法 | ❌ 无 | ❌ **不允许** | 云端 precheck 拦截 |
| BGM 用法 | ✅ 有 | ❌ **严禁** | 字幕条会跟着音乐进度走，毫无意义 |
| 无 audio | ✅ 有 | ❌ **绝对禁止** | LLM 严禁主动生成 subtitles（schema dependencies 强约束） |

### 3.4 字幕规则补充（仅配音用法适用）

1. **严禁创作模式或 BGM 模式生成 `subtitles` 字段**
   - 用户要求"加字幕条但不要配音" → 一句话拒绝："字幕需要跟配音对齐，没配音的字幕会乱跑"

2. **配音模式下字幕必须完整覆盖音频**
   - 最早 `start` ≈ 0
   - 最晚 `end` ≈ audio 时长（容差 ±0.5s）
   - 字幕之间不允许 > 3s 空白

3. **字幕文本要求**
   - 必须严格来自用户提供的 SRT，**严禁 LLM 自己改写、压缩、扩展**
   - 一条 ≤ 25 字（中文）/ 15 词（英文）
   - 不允许包含 emoji（部分前端字幕样式不支持）

## 4. 创作模式默认配 BGM（强制）

**创作模式启动时**，LLM **默认必须**给视频配上 BGM——除非用户主动说"不要 BGM"。

**目的**：纯图文视频如果完全静音，观感会很冷清；配一段轻量 BGM 能瞬间提升专业度。

### 4.1 自动选源策略

1. **不要询问用户偏好风格**（避免增加用户决策成本）
2. LLM 根据视频主题/风格**自动**从 `templates/bgm/` 内置库中选一首最匹配的
3. 直接把选中的文件路径写到 `project.json`，并设上 `loop: true, fadeIn: 1, fadeOut: 2`
4. 视频生成完成时，**简短告知用户**："我给视频配了一段 {风格} 风的 BGM，如果想换可以告诉我"

### 4.2 用户主动拒绝 BGM

用户说"不要 BGM" / "静音" / "不加音乐"时：
- 不写 audio 字段
- 视频静音播出（仍然合法）

**详见** [`../templates/bgm/bgm-catalog.md`](../templates/bgm/bgm-catalog.md) —— 完整 BGM 库速查与风格匹配表。

## 严禁清单

- ❌ 让用户回答"你想用什么模式"这种开放式问题（创作 vs 口播 由是否提供 SRT 自然推断）
- ❌ 配音模式下漏掉 `subtitles` 字段
- ❌ BGM 模式下生成 `subtitles` 数组
- ❌ 创作模式下 LLM 主动编造字幕内容
- ❌ 创作模式不配任何音频也不告知用户（除非用户明确拒绝 BGM）
