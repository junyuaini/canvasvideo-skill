# 节奏与动态密度规则（Timing Rules）

> 视频画面动态密度的 4 条强制门槛 + 时长建议。
> **本文档是 hard rule 单一来源**，被以下流程引用：
> - 子流程：video_design_guide.md 步骤 8（节奏设计）、步骤 10（时间轴）
> - 自检：references/selfcheck-rules.md（L0/L4 检查表）
> - 主流程：SKILL.md「第二次交互」严禁清单

---

## 问题背景

创作模式没有 SRT 钉时间，LLM 倾向把 60s 视频均分成 5×12s，并把所有组件挤在区域开头几秒，剩下十几秒画面死寂——观感像幻灯片，不像视频。

**核心原则**：**视频是动态艺术，画面不能长时间停在一个静止帧**。本规则强制保证画面动态密度。

---

## 4 条强制门槛

### 门槛 1：末组件停留时长（按模式区分）

**规则**：每个区域内"最后一个组件 start"到"区域结束（下一区域第一个 start，或 duration）"的间隔，**按模式有不同上限**：

| 模式 | 末组件停留上限 | 推荐区间 |
|------|---------------|---------|
| **创作模式** | **≤ 1 秒** | 0.3-1.0 秒 |
| **口播模式** | **≤ 2 秒** | 0.5-1.8 秒（中位数 ≈ 1.5 秒） |

**为什么按模式分**：
- 创作模式画面没有配音锚点，全靠组件动态推进，停留稍长就会让人觉得"卡住了"
- 口播模式有配音引导用户注意力，末段允许略长的停留来沉淀信息

**真实示例参考**：示例-案例分享型-1分钟口播.json 实测——10 个区域中 7 个 ≤ 2 秒、最大 2.8 秒、中位数 1.8 秒（口播模式数据）。

**违例**：
```
P3 区：start=24, end=36, duration=12s
  P3-001 标题  start=24
  P3-002 卡片  start=26
  P3-003 卡片  start=27
  ❌ 最后一个 start=27, 区域 end=36，静止 9 秒
```

**修复方案（任选）**：
- A. 把区域拆短：P3 改为 24-30s
- B. 在尾段加收尾组件（Image/Graphic/Shock/Quote），让 start ≥ end - (创作模式) 1s / (口播模式) 2s
- C. 把已有组件 start 后移，让最后一个 start 接近 end - 上限

---

### 门槛 2：组件出场最大间隔（按模式区分）

**规则**：同一区域内任意两个相邻组件（按 start 排序）的 start 时差，**按模式有不同上限**：

| 模式 | 相邻组件间隔上限 | 推荐区间 |
|------|----------------|---------|
| **创作模式** | **≤ 1 秒** | 0.3-1.0 秒 |
| **口播模式** | **≤ 3 秒** | 0.8-2.5 秒 |

**为什么按模式分**：
- 创作模式没有配音锚点，组件之间间隔超过 1 秒就会有"等待感"，让观众跑神
- 口播模式有配音引导，组件可以跟随讲解节奏稍长间隔出现

**违例**：
```
P2 区（创作模式）：标题 start=12, 卡片 start=14 → 间隔 2s ❌（创作 ≤ 1s）
P2 区（口播模式）：标题 start=12, 卡片 start=18 → 间隔 6s ❌（口播 ≤ 3s）
```

**修复方案**：
- A. 在两个组件之间补一个过渡组件（Badge / 副标题 Text / 装饰 Image），把大间隔切成多个小间隔
- B. 把后一个组件 start 提前，让间隔落到上限内
- C. 创作模式下，如果实在要长间隔，把这段拆成两个区域

---

### 门槛 3：区域时长上限（按模式区分）

**规则**：单一区域 `duration` **按模式有不同上限**：

| 模式 | 区域时长上限 | 推荐区间 |
|------|------------|---------|
| **创作模式** | **≤ 5 秒** | 3-5 秒 |
| **口播模式** | **≤ 8 秒** | 4-8 秒 |

超出上限必须拆分为更短区域。

**为什么按模式分**：
- 创作模式没有 SRT 锚点，区域超过 5 秒就很难填满紧凑节奏，必然出现长静止
- 口播模式区域时长跟随讲解段落自然延展，8 秒以内仍可保持注意力

**推荐区域数**：

| 视频时长 | 创作模式（≤5s/区） | 口播模式（≤8s/区） |
|---|---|---|
| 30s  | ≥ 6 个 | ≥ 4 个 |
| 60s  | ≥ 12 个 | ≥ 8 个 |
| 120s | ≥ 24 个 | ≥ 15 个 |
| 180s | ≥ 36 个 | ≥ 23 个 |

---

### 门槛 4：创作模式平均出场密度 ≥ 0.6 个/秒

**规则**：创作模式下，`组件总数 / 视频总时长 ≥ 0.6`。

**适用**：**仅创作模式**

**真实示例参考**：案例分享型 ≈ 0.87，产品演示型同量级。

**违例**：
```
60s 视频，组件总数 = 18 → 密度 = 0.3 ❌（低于 0.6）
```

**修复方案**：增加 ImageComponent / GraphicComponent / 子组件，或缩短视频时长。

---

## 操作建议（写设计时实践）

1. **每个区域必须有"收尾组件"**：在 `end - 2s` 之内出现一个新组件
2. **不要追求"区域整数化"**：6s/8s/10s/12s 这样的不等长区域才是好节奏
3. **60s 视频建议 ≥ 6 个区域**：宁可拆细也不要单一区域 > 15s
4. **组件之间用 1.5-3s 间隔密集出场**：大于 3s 一定要补过渡组件

## settings 三个动画参数（必须设置，有上限）

视频开始/结束/区域切换的三个过渡时长，影响整体节奏：

| 字段 | 含义 | 推荐值 | 上限 | 严禁 |
|---|---|---|---|---|
| `preFullViewDuration` | 视频**开始前**的全景预览时长（秒） | **0.4** | **0.6** | > 0.6 会让用户等太久 |
| `postFullViewDuration` | 视频**结束后**的全景回收时长（秒） | **0.4** | **0.6** | > 0.6 会让结尾拖沓 |
| `contentZoomRatio` | 视频播放区域的内容缩放比例 | **0.9** | — | 不要写 1.0（会贴边）；不要 < 0.85（内容会偏小） |

**完整 settings 示例**：
```json
"settings": {
  "autoPlay": false,
  "loop": false,
  "minScale": 0.01,
  "maxScale": 5,
  "ease": 0.08,
  "contentZoomRatio": 0.9,
  "preFullViewDuration": 0.4,
  "postFullViewDuration": 0.4
}
```

**严禁**：
- `preFullViewDuration / postFullViewDuration > 0.6`
- `contentZoomRatio = 1.0` 或 `< 0.85`
- 漏掉这三个字段（系统 fallback 可能不符合预期）
