# 布局与组件位置规则（Layout Rules）

> viewport / canvas / regions 坐标公式 + 组件 Y 坐标递增 + 尺寸计算 + 文字对比度 + 布局多样化。
> **本文档是 hard rule 单一来源**，被以下流程引用：
> - 子流程：video_design_guide.md 步骤 4（区域规划）、步骤 6（布局位置）、步骤 9（视觉样式）
> - 自检：references/selfcheck-rules.md（L0/L2 检查表）

## 1. viewport 与 regions 布局

### 1.1 viewport 尺寸

- **默认**：用户未提尺寸要求时，使用 `viewport: { width: 780, height: 585 }`（4:3 比例）
- **用户指定比例**：用户主动提比例时，按下方映射表换算像素尺寸
- **严禁**：不要主动问用户"要什么比例"，除非用户自己提

| 比例 | 用途 | viewport |
|---|---|---|
| 4:3（默认）| 通用横屏、PPT 风格 | `width: 780, height: 585` |
| 16:9 | 宽屏视频、电影感 | `width: 780, height: 439` |
| 1:1 | 正方形、社交媒体 | `width: 780, height: 780` |
| 9:16 | 竖屏、短视频 | `width: 439, height: 780` |
| 3:4 | 竖屏、小红书 | `width: 585, height: 780` |

### 1.2 regions 坐标公式

**规则**：所有区域按"从左到右一行 4 个"排布，超过 4 个换到下一行。

```
x = 120 + col × viewport.width
y = 50  + row × viewport.height

其中 row = Math.floor((index - 1) / 4)，col = (index - 1) % 4
```

**以默认 780×585 为例**：

| 列序 | x 坐标 | | 行序 | y 坐标 |
|---|---|---|---|---|
| 第 1 列 | 120 | | 第 1 行 | 50 |
| 第 2 列 | 900 | | 第 2 行 | 635 |
| 第 3 列 | 1680 | | 第 3 行 | 1220 |
| 第 4 列 | 2460 | | 第 4 行 | 1805 |

### 1.3 canvas 尺寸公式

**canvas 是覆盖所有区域的最小画布**：

```
canvas.width  = 最右侧区域 x + viewport.width  + 60  // 右边距 60
canvas.height = 最下方区域 y + viewport.height + 65  // 底边距 65
```

**以默认 780×585 为例**：

| 区域数 | 行数 | 最右 x | 最下 y | 推荐 canvas |
|---|---|---|---|---|
| 1-4 | 1 行 | 2460 | 50 | `width: 3300, height: 700` |
| 5-8 | 2 行 | 2460 | 635 | `width: 3300, height: 1285` |
| 9-12 | 3 行 | 2460 | 1220 | `width: 3300, height: 1870` |
| 13-16 | 4 行 | 2460 | 1805 | `width: 3300, height: 2455` |

**完整示例（10 个区域，2 行）**：
```json
"viewport": { "width": 780, "height": 585 },
"regions": [
  { "name": "P1", "x": 120,  "y": 50  },
  { "name": "P2", "x": 900,  "y": 50  },
  { "name": "P3", "x": 1680, "y": 50  },
  { "name": "P4", "x": 2460, "y": 50  },
  { "name": "P5", "x": 120,  "y": 635 },
  { "name": "P6", "x": 900,  "y": 635 },
  { "name": "P7", "x": 1680, "y": 635 },
  { "name": "P8", "x": 2460, "y": 635 },
  { "name": "P9", "x": 120,  "y": 1220 },
  { "name": "P10","x": 900,  "y": 1220 }
],
"canvas": { "width": 3300, "height": 1285 }
```

**严禁**：
- 写每行 5 个或 6 个（横向太长，跨区域过场动画会拖沓）
- 写每行 3 个（与已有真实示例不一致，破坏视觉节奏）
- 行间距不固定（默认 780×585 时行间距 = 585）

## 2. 组件 Y 坐标：必须连续递增，禁止重叠

**规则**：同一区域内，所有组件的 `position.y` 必须**严格递增**，后一个组件的 `y` 必须 ≥ 前一个组件的 `y + h + 间距`（建议间距 10-20px）。

**错误示例**（y 坐标回退，组件堆叠）：
```json
{ "id": "P1-005", "position": { "y": 535, "h": 60 } },  // bottom=595
{ "id": "P1-006", "position": { "y": 0,   "h": 80 } },  // ❌ y=0 < 595，重叠！
```

**正确示例**：
```json
{ "id": "P1-005", "position": { "y": 535, "h": 60 } },  // bottom=595
{ "id": "P1-006", "position": { "y": 605, "h": 80 } },  // ✅ y=605 ≥ 595+10
```

**自检公式**：生成完一个区域的所有组件后，按 `y` 排序，检查 `sorted[i].y ≥ sorted[i-1].y + sorted[i-1].h + 10`。

**严禁**：
- 同一区域内出现两个组件 `y` 相同或后一个 `y` 小于前一个的 `bottom`
- 为了"对齐美观"故意把后面组件的 `y` 设成 0

## 3. 组件尺寸：w/h 必须与内容匹配，禁止溢出

**规则**：每个组件的 `position.w` 和 `position.h` 必须**显式填写**，且尺寸要与实际内容匹配。

### 3.1 文字类组件高度估算公式

```
h = 上边距 + 行数 × 行高 + 下边距
```

| 组件类型 | 字号 | 行高 | 左右边距 | 上下边距 |
|---------|------|------|---------|---------|
| TitleComponent | 36-48px | 1.3×字号 | 20px | 10px |
| TextComponent | 18-24px | 1.5×字号 | 20px | 10px |
| QuoteComponent | 20-28px | 1.6×字号 | 24px | 12px |
| BadgeComponent | 14-18px | 1.4×字号 | 12px | 6px |
| CardComponent | 16-20px | 1.5×字号 | 16px | 10px |

### 3.2 宽度约束

- 区域内组件的 `w` 不应超过 `viewport.width - 40`（左右各 20px 边距）
- 默认 780px 视口下，文字组件 `w ≤ 740`

### 3.3 高度约束（防止溢出）

- 一个区域内所有组件的 `h` 之和 + 间距之和 必须 `≤ viewport.height - 20`
- 默认 585px 视口下，组件总高度不应超过 565px
- **严禁**：组件底部 `y + h > viewport.height`

### 3.4 错误/正确对比

```json
// ❌ 错误：3 行文字但 h=40 只够 1 行
{
  "content": "第一行\n第二行\n第三行",
  "position": { "x": 20, "y": 100, "w": 700, "h": 40 }
}

// ✅ 正确：h=110 = 上边距10 + 3行×27 + 下边距10 + 缓冲10
{
  "content": "第一行\n第二行\n第三行",
  "position": { "x": 20, "y": 100, "w": 700, "h": 110 }
}
```

**自检**：每个文字组件 `h ≥ 行数×行高+边距`；最后一个组件 `y + h ≤ viewport.height - 10`。

**严禁**：
- `position.w` 或 `position.h` 不填写（必须显式写）
- 文字组件 `h` 小于实际内容所需高度
- 组件堆叠总高度超过 `viewport.height`
- 把 `h` 写死成固定小值（如所有组件 h=50）

## 4. 文字与背景对比度（强制）

| 场景 | 背景特征 | 文字色要求 |
|------|---------|-----------|
| 白底/浅底 | `white` 主题，`#FFFFFF`~`#F3F4F6` | 文字必须用深色：`#111827`/`#1F2937`/`#374151` |
| 黑底/深底 | `black` 主题，`#000000`~`#1F2937` | 文字必须用亮色：`#FFFFFF`/`#F3F4F6`/`#E5E7EB` |
| 图片上方 | 背景图、渐变图 | 文字必须加 `textShadow` 或 `backdropFilter`，且与图片主色调差异 ≥ 50% |
| 彩色背景块 | Badge/Card/强调色背景 | 背景亮度 > 50% 用深字；< 50% 用亮字 |

**亮度快速判断**：
- 白/浅灰/浅蓝/浅绿 → 文字用 `#111827`
- 黑/深灰/深蓝/深紫 → 文字用 `#FFFFFF`
- 彩色（橙/红/青）→ 文字用黑或白，取决于哪个更醒目

**严禁**：
- 文字色与背景色属于同一色系且亮度接近（如浅灰底 + 更浅灰字）
- 图片上方文字不加 `textShadow` 或 `backdropFilter`

## 5. 区域布局多样化（禁止复制粘贴）

**规则**：相邻区域（如 P1→P2→P3）的组件结构、类型顺序、排列方式**不能完全相同**。必须至少有 3 种不同的布局模板交替使用。

**错误示例**：
```
P1-P10 全部 = Title → Text → Badge → Aggregate(Image+Shock) → Text → Shock → Quote → Card
❌ 复制粘贴
```

**正确示例**：
```
P1（A）: Title → Text → Aggregate(Image+Shock) → Text → Shock → Quote
P2（B）: Title → Badge → Image → Text → Card → Quote
P3（C）: Title → Text → Image → Shock → Quote → Card
P4（A）: 与 P1 相同但间隔 3 个区域 ✅
```

**布局变化维度**（至少变化 2 个以上）：
1. **组件数量**：5/7/9 个错落
2. **组件类型组合**：用 Card vs 不用；用 Graphic vs 用 Image
3. **排列顺序**：图片放上面 vs 中间；Quote 放开头 vs 结尾
4. **有无 Aggregate**：图片直接放顶层 vs 包在 Aggregate 里
5. **文字密度**：以图为主（1 图+2 文）vs 以文为主（4 文+1 图）

**自检**：把每个区域的"组件类型序列"列出来对比，任意两个相邻区域的类型序列完全相同必须修改其中一个。
