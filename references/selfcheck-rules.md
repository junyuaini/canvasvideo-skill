# 自检规则（Self-Check Rules）

> 生成完 `project.json` 后必须进行的 5 级自检（L0~L4）+ 设计原则检查 + 丰富度评分。
> **本文档是 hard rule 单一来源**，被以下流程引用：
> - 子流程：video_design_guide.md 步骤 11（自检报告）
> - 主流程：SKILL.md「第二次交互」严禁清单、第四次交互打包前

---

## 自检规则总览

| 级别 | 含义 | 不通过的处理 |
|------|------|-------------|
| L0 | **致命错误**（结构性、必填字段） | 必须修复，否则不允许打包 |
| L1 | **严重违规**（影响视频可用性） | 必须修复 |
| L2 | **中等建议**（视觉/可读性） | 强烈建议修复 |
| L3 | **轻微优化**（统一性、美观） | 建议修复 |
| L4 | **丰富度**（防止 low 视频） | 必须修复（不达标会生成幻灯片感视频） |

---

## L0 检查（致命错误）

| 检查项 | 通过条件 | 规则来源 |
|--------|---------|---------|
| 组件 start ≥ 字幕 start | 所有组件 start 时间正确 | mode-rules.md |
| 组件 end ≤ 下一区域 start | 无时间重叠 | timing-rules.md |
| 相邻区域无时间重叠 | P1(0-12), P2(12-24)... | timing-rules.md |
| 图片切换 end=start | 无图片切换重叠 | — |
| **非 AggregateComponent 必须有 customStyle 字段** | 所有非 Aggregate 组件含 `customStyle`，无样式时也要写 `{}` | components-catalog.md §"全局硬约束" |
| TitleComponent.customStyle 含 `level{N}` 嵌套 | 默认 level1 | components-catalog.md |
| TextComponent.customStyle 含 `paragraph` 或 style 嵌套 | 默认 paragraph | components-catalog.md |
| 平铺类组件必填字段无空值 | Badge/Card/Shock 等已写齐 | API: /api/component/spec |
| **字幕与音频共生** | 创作=无 audio 无 subtitles；口播=audio+subtitles 都有 | mode-rules.md §3 |
| **末组件停留 ≤ 2s** | 每个区域末组件→end ≤ 2 秒 | timing-rules.md 门槛 1 |
| **创作模式单区域 ≤ 15s** | 创作模式所有区域 ≤ 15 秒 | timing-rules.md 门槛 3 |
| **组件尺寸与内容匹配（无溢出）** | 文字 h ≥ 行数×行高+边距；y+h ≤ viewport.height-10 | layout-rules.md §3 |
| **组件 Y 坐标连续递增（无重叠）** | sorted[i].y ≥ sorted[i-1].y + sorted[i-1].h + 10 | layout-rules.md §2 |

## L1 检查（严重违规）

| 检查项 | 通过条件 | 规则来源 |
|--------|---------|---------|
| 区域组件数 ≥ 最小要求 | 每个区域 ≥ 2 个组件 | richness-rules.md |
| 无与字幕重复文字 | 字幕已说的文字不要再放在屏幕上 | mode-rules.md |
| 标题→副标题→图片顺序 | 视觉层级正确 | layout-rules.md |
| 多逻辑分组用 Aggregate | 多列对比、4 卡片网格等用 AggregateComponent | components-catalog.md |

## L2 检查（中等建议，对比度强约束）

| 检查项 | 通过条件 | 规则来源 |
|--------|---------|---------|
| 浅色背景→深色文字 | 文字色 #111827 等 | layout-rules.md §4 |
| 深色背景→亮色文字 | 文字色 #FFFFFF 等 | layout-rules.md §4 |
| 文字叠图→shadow / backdropFilter | 所有图片上方文字加防护 | layout-rules.md §4 |
| 彩色背景块→对比鲜明 | 橙底用黑字，蓝底用白字 | layout-rules.md §4 |
| 无低对比度配色 | 浅灰底+更浅灰字 严禁 | layout-rules.md §4 |
| 主体图片 borderRadius=0 | — | — |
| 长图高度 ≥ 400px | — | — |
| Badge 文字 > 4 字 → 宽度 ≥ 120px | — | — |

## L3 检查（轻微优化）

| 检查项 | 通过条件 |
|--------|---------|
| 阴影统一性 | 同视频阴影 box-shadow 风格一致 |
| 圆角统一性 | borderRadius 风格一致 |
| 渐变方向 | 相邻区域渐变方向有变化 |

## L4 丰富度检查（不通过必须重写步骤 7-9）

> 详细规则见 [`richness-rules.md`](./richness-rules.md)。

| 检查项 | 通过条件 |
|------|---------|
| 组件类型覆盖率 | ≥ 60%（用满 ≥ 6 种组件类型） |
| 同一类型连续使用 | ≤ 3 个 |
| 每区域非纯文字组件 | 每个区域 ≥ 1 个 |
| 素材清单实现率 | 100%（清单中所有素材都有 ImageComponent 引用） |
| Graphic diagram 多样性 | ≥ 3 种（视频时长 ≥ 60s） |
| 同区域同类组件配色差异 | 连续 ≥ 3 个同类组件，至少 2 种基调 |
| **节奏 4 条门槛** | 详见 [`timing-rules.md`](./timing-rules.md) |

## 设计原则检查

| 原则 | 通过条件 |
|------|---------|
| 一个区域一个焦点 | 每个区域聚焦 1 个核心信息（金句/对比/步骤/数据/按钮） |
| 信息分层 | 大标题→中副标题→小内容 |
| 用对比制造冲击 | P 区设计利用红绿/明暗/大小对比 |
| 用真实替代抽象 | 步骤截图、成果截图、真实证言 |
| 情绪曲线合理 | 高→中低→中→中高→中（按视频结构） |

---

## 丰富度评分（自检用，满分 70）

| 维度 | 0 分 | 5 分 | 10 分 |
|---|---|---|---|
| 组件类型多样性 | < 4 种 | 5-6 种 | ≥ 7 种 |
| 同类连续 | 出现 5+ 个连续 | 4 个连续 | ≤ 3 个连续 |
| 非文字组件 | 有区域 0 个 | 每区域至少 1 个 | 多种穿插 |
| 素材实现率 | < 50% | 50-99% | 100% |
| Graphic 多样性 | 0-1 种 | 2 种 | ≥ 3 种 |
| 同类配色差异 | 100% 同色 | 部分差异 | 渐变/对比/多色 |
| **动态密度（节奏）** | 有区域静止 ≥ 5s | 有区域静止 2-5s | 全部 ≤ 2s |

**评分要求**：
- < 50 分 → 视频会很 low，**必须重写**
- 50-64 分 → 中等，**建议优化**
- ≥ 65 分 → 接近真实示例水平，**可以打包上传**

## 填写示例（真实自检报告）

LLM 写 design.md 步骤 11 时，按下面格式填写完整自检表，**每一格必须给真实值，禁止全部填"通过"**。

### L0 检查（节选示例）

```
| 末组件停留 ≤ 2s            | ✅ | P1=1.5, P2=1.0, P3=0.8, P4=1.2, P5=0.5, P6=1.0（不等长，0.5-1.5s 区间） |
| 创作模式单区域 ≤ 15s       | ✅ | 最长 P3=12s ≤ 15s |
| 组件尺寸与内容匹配         | ✅ | 所有文字组件 h ≥ 行数×行高+边距；最末组件 y+h ≤ 575 |
| 组件 Y 坐标连续递增        | ❌ | P1 区 P1-006 y=0 < P1-005 bottom=595，需要把 P1-006 移到 y=605 |
```

### L4 丰富度检查（节选示例）

```
| 组件类型覆盖率   | ✅ | 用了 7/10 种 = 70%（Title/Text/Image/Card/Badge/Shock/Graphic） |
| 同一类型连续    | ✅ | 最长连续 3 个（P3 三张支柱 Card） |
| 每区域非纯文字   | ✅ | P1=Shock, P2=Graphic, P3=Card+Image, P4=Aggregate(Card), P5=Graphic |
| 素材清单实现率   | ❌ | 清单 8 个，project.json 引用 0 个 → 必须补 8 个 ImageComponent |
| Graphic 多样性  | ❌ | 只用了 flow × 2 → 必须把 P3 改 pyramid、P4 改 matrix |
| 同区域同类配色   | ❌ | P3 三张 Card 用同一蓝紫渐变 → 必须改成"渐进色阶"或"对比基调" |
| 节奏 4 条门槛   | 详见 timing-rules.md | P1=1.5s, P2=1.8s, ... 详见时间轴 |
```

### 丰富度评分示例

```
| 维度          | 实际得分 |
| 组件类型多样性 | 10（7 种） |
| 同类连续      | 10（最长 3） |
| 非文字组件    | 10 |
| 素材实现率    | 0（清单 8 个，引用 0 个） |
| Graphic 多样性 | 5（只用了 2 种）|
| 同类配色差异   | 0 |
| 动态密度      | 10 |
| **总分**     | **45 / 70** → < 50 分，必须重写（详见"评分要求"） |
```

## 严禁清单（自检阶段）

- ❌ 自检报告全部填"通过"（必须列出实际值）
- ❌ 跳过任何一级检查
- ❌ L0/L4 任何一条不通过仍上传
- ❌ 把"⚠️ 注意"或"⚠️ 建议"用在 L0/L4（必须明确通过/不通过）
