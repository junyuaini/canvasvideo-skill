# 自检规则

> 本地自检只做两项检查，其余交给云端。

---

## 本地自检（两项）

### 1. ID 格式检查

**格式**：`P{数字}-{三位数字}`，如 `P1-001`、`P3-005`、`P10-001`

| 正确 | 错误 |
|------|------|
| P1-001 | P1-1 |
| P2-005 | P2-05 |
| P3-010 | 003 |
| P10-001 | P1_001 |

### 2. ID 重复检查

所有组件 ID 必须全局唯一。

---

## 格式说明

- `P` 后跟数字表示区域编号（P1、P2、P3...）
- `-` 分隔符
- 三位数字表示组件序号（001、002、003...）
- 组件 ID 用于后续修改时的精确引用（如 "把 P3-004 替换成 xxx"）

---

真正的格式硬校验（schema、字段完整性）由云端 `/api/projects/validate` 在上传前完成。
