# 自检规则

> 视频生成后的自检检查表与评分标准。
> 
> **三段式流程**：原则指导 → 自由创作 → 兜底检查

---

## R1 自检层级

| 层级 | 检查项 | 通过条件 |
|------|--------|---------|
| L0 | 文件完整性 | output.zip 存在且可解压 |
| L1 | JSON 结构 | project.json 合法、字段完整 |
| L2 | 组件规范 | ID唯一、时间无重叠、Y递增 |
| L3 | 视觉丰富度 | 6条门槛 ≥4分 |
| L4 | 音频同步 | 配音模式：字幕覆盖音频±0.5s；BGM模式：无subtitles |

---

## R2 兜底检查（三级严重程度）

### Error（必须修复）

| 检查项 | 规则 | 修复方式 |
|--------|------|---------|
| 焦点缺失 | 每区域至少1个焦点组件 | 添加焦点组件（Shock/Title/Image） |
| 时间覆盖 | 组件时间必须覆盖区域时间 | 调整组件start/end |
| 语义不一致 | 组件内容与区域主题不匹配 | 修改组件内容 |
| ID重复 | 组件ID全局唯一 | 修改重复ID |
| 时间冲突 | 同一区域内组件时间重叠 | 调整时间轴 |

### Warning（建议优化）

| 检查项 | 规则 | 建议 |
|--------|------|------|
| 区域时长 | 口播≤8秒，创作≤8秒（建议5秒） | 时长过长时拆分区域 |
| 组件过多 | >8个组件/区域 | 检查是否必要，考虑拆分 |
| 组件过少 | <2个组件/区域（非极简模式） | 确认是否为极简设计 |
| 层级不清 | 字号/颜色无明显区分 | 区分主次 |

### Info（仅供参考）

| 检查项 | 规则 | 说明 |
|--------|------|------|
| 留白检查 | 组件不贴边，间距均匀 | 视觉舒适度 |
| 模式一致性 | 同视频内模式变化是否合理 | 避免单调 |
| 色彩丰富度 | 是否使用了多种颜色 | 视觉吸引力 |

---

## R3 评分标准

| 分数 | 状态 |
|------|------|
| 0 | 未通过任何层级 |
| 1 | 通过 L0 |
| 2 | 通过 L0+L1 |
| 3 | 通过 L0+L1+L2（无Error） |
| 4 | 通过 L0+L1+L2+L3 |
| 5 | 通过全部层级 |

**目标**：≥ 4 分（及格线）

**处理原则**：
- 有 Error → 必须修复后才能继续
- 有 Warning → 建议修复，但不阻塞
- 有 Info → 仅供参考

---

## R4 自检流程

1. 运行 `scripts/validate.js` 自动检查 L0~L2
2. 运行 `scripts/selfcheck.js` 检查兜底项（Error/Warning/Info）
3. 对照 `rules/05-richness.md` 检查 L3
4. 对照 `rules/02-mode.md` §R4 检查 L4
5. 汇总评分，低于 4 分需修复
