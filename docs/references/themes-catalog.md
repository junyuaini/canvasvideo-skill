# 主题目录（CanvasVideo Skill 知识库）

> CanvasVideo 当前**只支持两种主题**：`white`（极简白）/ `black`（沉浸黑）。
> ⚠️ 不支持自定义主题，LLM 不允许承诺"按品牌色定制"或"做成其他主题"。

---

## 两种主题速记

| schema 字段值 | 用户面前的名字 | emoji | 一句话定位 |
|---|---|---|---|
| `"white"` | **极简白** | 🤍 | 白底 + 多色光晕，深字清晰，**商务演示风** |
| `"black"` | **沉浸黑** | 🌌 | 深紫蓝底 + 霓虹高亮，沉浸感强，**科技潮流风** |

> **跟用户对话时**用"极简白 / 沉浸黑"。**写 project.json 时**必须用 schema 字段值 `"white"` / `"black"`。

---

## ⚠️ 强制规则（不可违反）

1. `project.json` 的 `theme` 字段**只能填 `"white"` 或 `"black"`**（schema 枚举强约束）
2. **不允许伪造其他主题名**（如 `"colorful"` / `"company-brand"`）：枚举校验会直接拒绝
3. **不允许使用 `theme` 字段以外的全局主题机制**（v1.4 起强制 customStyle）
4. 用户要求"换品牌色 / 用我们公司颜色 / 渐变金色 / 粉色主题"等：**只能在极简白/沉浸黑二选一确定后，通过 `customStyle.color/textColor/background` 改组件级配色实现**，不要试图改 theme

---

## 主题影响范围

theme 不影响组件样式（组件样式由 `customStyle` 控制）。theme 只影响：

- 视频区背景渐变
- 控制面板（播放栏、按钮、组件导航）
- 字幕样式
- 系统 UI（全景按钮、ID 按钮、进度条）

所以：**选好 theme 之后，每个组件的 customStyle 字段必须通过 API 查询后填写**（`POST /cv/api/component/spec/batch`），禁止凭记忆编字段名。

⚡ **必须调 batch API**——不放宽这条硬规则。

## 选型决策表

| 用户主题/场景 | 推荐 |
|---|---|
| 产品介绍、商务演示、品牌宣传、教学科普、案例分享、API 文档、工具演示 | **white** |
| 代码演示、AI/区块链/加密、游戏潮流、数据看板、品牌发布、年度盘点 | **black** |
| 用户没明确要求 | **white**（默认） |

## 与用户沟通的标准话术

### ⭐ 主题二选一（必问，除非用户首条消息已明确选了）

```
当前 CanvasVideo 提供两种风格主题（仅支持这两种）：

🤍 极简白（white）
  - 白底 + 多色光晕，深字清晰
  - 适合：商务演示、产品介绍、教学科普、案例分享

🌌 沉浸黑（black）
  - 深紫蓝底 + 霓虹高亮，沉浸感强
  - 适合：技术分享、AI/代码、数据看板、品牌发布、潮流内容

请选 1 还是 2？也可以告诉我视频主题，我帮你推荐。
```

### 用户要求自定义主题（品牌色 / 橙色 / 粉色等）

```
当前 CanvasVideo 只支持「极简白」和「沉浸黑」两种主题，暂不支持自定义。
不过你提到的颜色可以通过组件级 customStyle 实现，最终视觉效果接近。

要不我们用「极简白」打底，关键组件（标题/按钮/数据高亮）用你的品牌色？
```

### 用户坚持完全自定义

```
理解你的需求。当前版本只支持「极简白 / 沉浸黑」两选一。
如果今天必须出片，建议先二选一打底，后续主题能力升级后可以再迭代一版。要继续吗？
```

## 常见错误自检

| 错误写法 | 后果 | 修复 |
|---|---|---|
| `"theme": "default"` / `"Custom"` / 任意非枚举值 | schema 校验拒绝 | 改成 `"white"` 或 `"black"` |
| `"theme": null` / 不写 theme 字段 | 视频加载报错或背景渲染异常 | 强制写 `"white"` 或 `"black"` |
| `"white"` 主题但组件文字用浅色 | 文字看不见 | 浅底用深字 |
| `"black"` 主题但组件文字用 `#000000` | 文字看不见 | 黑底用浅色文字 |
