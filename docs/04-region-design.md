# 步骤4：区域设计

> 前置步骤：[步骤3：生成骨架JSON](03-skeleton-build.md)
> 下一步：[步骤5：生成区域JSON](05-region-build.md)

---

## 目标

为单个区域设计组件、布局、样式和时间轴。

---

## 输入

| 来源 | 说明 |
|------|------|
| 骨架配置 | `skeleton.json` 中该区域的信息 |
| 上一步产出 | 前序区域的 design-P{n}.md（如有） |
| 引用规则 | `references/layout-rules.md`、`references/components-catalog.md` |

---

## 操作

### 第 1 步：读取模板

读取 `templates/artifacts/design-region.md`，了解输出格式。

### 第 2 步：确定区域内容

根据骨架设计中的区域主题，编写文案：

| 元素 | 内容 | 字数限制 |
|------|------|----------|
| 主标题 | 区域核心信息 | ≤ 15 字 |
| 副标题 | 补充说明 | ≤ 30 字 |
| 正文 | 详细内容 | ≤ 100 字 |
| 标签 | 关键词 | ≤ 8 字 |

### 第 3 步：设计布局

用 ASCII 绘制布局：

```
┌─────────────────────────────┐
│                             │
│     【主标题】               │
│                             │
│  ┌─────────────────────┐   │
│  │    【核心内容】      │   │
│  └─────────────────────┘   │
│                             │
│       [标签]                │
│                             │
└─────────────────────────────┘
```

### 第 4 步：选择组件

| 组件类型 | 适用场景 | 数量限制 |
|----------|----------|----------|
| TitleComponent | 主标题 | 每区域 1 个 |
| TextComponent | 正文 | 每区域 ≤ 2 个 |
| ShockComponent | 强调数据 | 每区域 ≤ 1 个 |
| BadgeComponent | 标签 | 每区域 ≤ 2 个 |
| ImageComponent | 图片 | 每区域 ≤ 1 个 |

**原则**：单个区域组件总数 ≤ 5 个

### 第 5 步：设计时间轴

**使用全局绝对时间**（基于骨架设计中的区域时间范围）：

假设 P1 的时间范围是 0-10s：

| 组件 | start | end | 说明 |
|------|-------|-----|------|
| P1-001 | 0 | 10 | 全程显示 |
| P1-002 | 2 | 10 | 延迟出现 |

**原则**：
- 组件时间使用**全局绝对时间**（不是相对时间）
- 组件 start ≥ 区域 start
- 组件 end ≤ 区域 end

### 第 6 步：生成设计文档

```markdown
# 区域设计：P1 — {主题}

## 内容规划
- 主标题："..."
- 副标题："..."

## 布局设计
```
[ASCII 图]
```

## 组件清单
| ID | 类型 | 内容 | 位置 |

## 时间轴
| 组件 | start | end |
```

### 第 7 步：保存文件

```js
fs.writeFileSync(
  path.join(workdirRoot, skillProjectId, `design-${regionName}.md`),
  designContent
);
```

---

## 产出

| 文件 | 路径 | 说明 |
|------|------|------|
| design-P{n}.md | `{workdir}/{skillProjectId}/design-P{n}.md` | 区域设计文档 |

---

## 自检

- [ ] 组件总数 ≤ 5 个
- [ ] 每个组件有明确的 ID
- [ ] 位置坐标在 viewport 范围内
- [ ] 时间轴无重叠冲突
