# 步骤2：骨架设计

> 前置步骤：[步骤1：初始化](01-init.md)
> 下一步：[步骤3：生成骨架JSON](03-skeleton-build.md)

---

## 目标

确定视频整体结构：区域数、时长分配、主题风格。

---

## 输入

| 来源 | 说明 |
|------|------|
| 上一步产出 | `design-skeleton.md`（如有迭代）|
| 用户输入 | 视频主题、时长、风格 |
| 引用规则 | `references/layout-rules.md`、`references/mode-rules.md` |

---

## 操作

### 第 1 步：读取模板

读取 `templates/artifacts/design-skeleton.md`，了解输出格式。

### 第 2 步：确定项目元信息

从用户输入提取：

| 字段 | 来源 | 默认值 |
|------|------|--------|
| 模式 | 用户是否提供 SRT/MP3 | 创作模式 |
| 主题 | 用户输入 | 必填 |
| 时长 | 用户输入 | 60 秒 |
| 风格 | 用户输入 | 科技感 |
| 音频用途 | 用户选择 | BGM |
| BGM 风格 | 用户选择或自动匹配 | tech-pulse |

### 第 3 步：区域划分

根据时长确定区域数，并分配**绝对时间范围**：

| 视频时长 | 建议区域数 | 每区域时长 |
|----------|-----------|-----------|
| 5-15 秒 | 1-2 个 | 5-10 秒 |
| 15-30 秒 | 2-3 个 | 8-15 秒 |
| 30-60 秒 | 3-5 个 | 10-20 秒 |
| 60-120 秒 | 5-8 个 | 10-20 秒 |

**时间分配原则**：
1. 每个区域有明确的**绝对时间范围**（如 P1: 0-10s, P2: 10-25s）
2. 区域时间连续，不重叠，不间隙
3. 单个区域不超过 20 秒
4. 区域设计时，组件时间使用**全局绝对时间**

**示例**：

| 区域 | 时间范围 | 主题 | 情绪 |
|------|----------|------|------|
| P1 | 0-10s | 开场钩子 | 好奇 |
| P2 | 10-25s | 痛点呈现 | 共鸣 |
| P3 | 25-45s | 方案展示 | 期待 |
| P4 | 45-60s | CTA | 冲动 |

### 第 4 步：视觉策略

- **主题**：从 `templates/themes/` 选择（black / white）
- **viewport**：根据视频比例确定（16:9 / 9:16 / 1:1）
- **canvas**：根据区域数计算宽度
- **BGM**：从 `templates/bgm/bgm-catalog.md` 选择

### 第 5 步：生成设计文档

按模板结构生成 `design-skeleton.md`：

```markdown
# 视频骨架设计：{主题}

## 步骤 0：项目元信息
| 字段 | 值 |
|------|-----|
| 模式 | 创作模式 |
| 主题 | AI时代如何学习 |
| 时长 | 60秒 |
| 风格 | 科技感 |
| 音频用途 | BGM |
| BGM 风格 | tech-pulse |

## 阶段一：区域划分
| 区域 | 时间范围 | 主题 | 情绪 |
|------|----------|------|------|
| P1 | 0-10s | 开场钩子 | 好奇 |
| P2 | 10-25s | 痛点呈现 | 共鸣 |
| P3 | 25-45s | 方案展示 | 期待 |
| P4 | 45-60s | CTA | 冲动 |

## 阶段二：视觉策略
- 主题：black
- viewport：1080x1920（9:16）
- canvas：5000x1300
- BGM：tech-pulse.mp3
```

### 第 6 步：保存文件

```js
const fs = require('fs');
const path = require('path');

fs.writeFileSync(
  path.join(workdirRoot, skillProjectId, 'design-skeleton.md'),
  designContent
);
```
